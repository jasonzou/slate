    Introduction

    This is the documentation for the AtoM 3 API! This document lists the endpoints that are used by the backend server to communicate with the frontend in the application.

    This example API documentation page was created with Slate.

    Authentication

    Most requests to the AtoM 3 backend requires an user to be authenticated. This can be done with a simple POST request to the /user/:user_name/login endpoint, with :user_name and :password parameter being supplied.

    The JSON that is returen will have a session key, which can be stored and used for other requests. Sessions will be expire after an hour (it can be configured in config.yml file)j

    To authorize, use this code:

    import atom3

api = atom3.authorize('atom3atom3atom3')

    # With shell, you can just pass the correct header with each request
curl "api_endpoint_here"
  -H "Authorization: atom3atom3atom3"

    const atom3 = require('atom3');

let api = atom3.authorize('atom3atom3atom3');

    Make sure to replace atom3 with your API key.

    AtoM 3 uses API keys to allow access to the API. You can register a new AtoM 3 API key at our developer portal.

    AtoM 3 expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: atom3atom3atom3

    You must replace atom3atom3atom3 with your personal API key.

    require 'kittn'

api = AtoM 3::APIClient.authorize!('meowmeowmeow')
api.kittens.get(2)

api = kittn.authorize('meowmeowmeow')

    const kittn = require('kittn');

let api = kittn.authorize('meowmeowmeow');
let max = api.kittens.get(2);

  "id": 2,

    Errors

    This error section is stored in a separate file in includes/_errors.md. Slate allows you to optionally separate out your docs into many files...just save them to the includes folder and add them to the top of your index.md's frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The kitten requested is hidden for administrators only.
    404 Not Found -- The specified kitten could not be found.
    405 Method Not Allowed -- You tried to access a kitten with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The kitten requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many kittens! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
